Project Overview

This project explores the design and validation of a low-cost, lightweight ISR fixed-wing UAV incorporating a FPV vision-based click-to-track capability under strict computational, mass, and manufacturability constraints. The primary objective is to demonstrate that stable target tracking and flight control can be achieved using minimal onboard processing and commercial off-the-shelf components, without reliance on high-performance embedded computers.The system integrates a lightweight vision tracker with a custom Kalman filter implemented on an ESP32 companion microcontroller, which smooths and predicts target coordinates prior to injection into the flight controller.

This architecture explicitly addresses vision and communication latency, reducing aggressive control inputs that would otherwise induce excessive ESC thermal loading, actuator wear, and degraded endurance. By prioritising smooth control behaviour, the design enables the use of lighter propulsion hardware and contributes to improved flight efficiency. UAV frame performance was evaluated using XFLR5 followed with manufacturing and material testing, allowing controlled assessment of CG testing with payload placement assessing impacts of vibration and mounting tests. Tracking system performance was evaulated using HITL - Python OPENCV with esp32 to assess latency effects, estimator behaviour, and closed-loop coordinate accuracy response. The results demonstrate that Kalman-based prediction significantly mitigates latency-induced phase lag, improving tracking stability within the operational limits imposed by low-cost telemetry and embedded hardware.

At extended distances, FPV video becomes the limiter, increased transmission delay and packet loss introduce phase lag between vision measurements and aircraft response, which directly degrades closed-loop tracking performance. One of the core motivations for the onboard Kalman filter is to mitigate this latency through prediction, extending the usable operating envelope before instability becomes dominant.

System Architecture & Data Flow

The following flow diagram illustrates the closed-loop control and estimation pipeline. By offloading vision processing to the Ground Station and state estimation to the ESP32, the system maintains high-frequency tracking despite the inherent latency of the telemetry link. 

                [Target]
                   ↓
                [FPV Camera]
                   ↓  (Analog video, low latency)
                [Ground Station]
                   ├─ Vision processing (OpenCV / CSRT)
                   ├─ Target detection (pixel coords)
                   ├─ Camera + attitude projection
                   └─ Target measurement (z_k)
                           ↓
                   MAVLink over Telemetry (LoRa)
                           ↓
                [Flight Controller]
                   ├─ Timestamped target update
                   ├─ State awareness (vehicle pose)
                   └─ UART forwarding
                           ↓
                [ESP32 Estimator]
                   ├─ Kalman predict step
                   ├─ Kalman update step
                   ├─ Latency compensation
                   └─ Smoothed target state (x̂_k)
                           ↓
                [Flight Controller]
                   ├─ Guidance / control
                   └─ Actuation (navigation)

Ground station

The role of the Ground Station is to receive the live FPV stream, detect and localise the target using OpenCV-based tracking, project image-space measurements into physical space using pixel frames and vehicle attitude, and transmit raw, timestamped measurements(co-ordinates) to the aircraft.

Output (Measurement Vector): 

                    zₖ = [x, y, (optional z), confidence, timestamp]

Flight Controller

The Flight Controller receives target measurements first and maintains full vehicle state awareness (position, attitude, and velocity). Acting as a central communication hub, it forwards measurements to the onboard estimator while retaining raw target information. This architecture enables fault tolerance, allowing the ESP32 to reset or fail without loss of target awareness.

ESP32 Kalman Filter

The ESP32 implements a continuous-time discrete Kalman filter responsible for: Smoothing noisy vision-based target measurements, Predicting target motion during telemetry dropouts, Compensating for vision + telemetry latency, Providing a time-aligned, low-jitter target state to the flight controller. 

                    [Kalman State Vector]
                             ↓
                    xₖ = [x, y, v_x, v_y]
                             ↓ [Predict Step]
                    x̂ₖ|ₖ₋₁ = F x̂ₖ₋₁
                    Pₖ|ₖ₋₁ = F Pₖ₋₁ Fᵀ + Q (This runs even when telemetry updates pause)
                             ↓ [Update Step]
                    Kₖ = P Hᵀ (H P Hᵀ + R)⁻¹
                    x̂ₖ = x̂ₖ|ₖ₋₁ + Kₖ (zₖ − H x̂ₖ|ₖ₋₁) (runs when GS data arrives)

Where:

R: Measurement noise covariance (derived from vision confidence)

Q: Process noise covariance (target manoeuvrability)

Latency Compensation and Time-Alignment of Target State

Vision processing and telemetry introduce unavoidable measurement latency. To address this, state estimation is explicitly decoupled from time alignment. Target measurements are first incorporated into the Kalman filter at their original timestamp, preserving estimator consistency and stability. The estimated state is then forward-projected to the current time using the estimated target velocity, ensuring that the Flight Controller receives a predictive, time-aligned target state rather than delayed observations. This significantly reduces phase lag and control jitter, enabling stable tracking at low update rates and during intermittent telemetry dropouts. 

                    x̂(t_now) = x̂(t_rx) + v̂ · Δt

Discussion

During testing, we analyzed the filter's performance regarding covariance tuning. By adjusting the Process Noise (Q) and Measurement Noise (R), we were able to achieve a balance between "responsiveness" and "smoothness." The discussion further explores how the Kalman filter maintains a target lock even when the OpenCV tracker briefly loses the object during a rapid banked turn of the UAV.

Aerodynamics and Stability Analysis

Aerodynamic Efficiency

The airframe geometry was optimized for high lift-to-drag (L/D) ratios to maximize flight endurance. Using Fusion 360's simulation tools, the wing's airfoil section was selected to provide stable lift at the low Reynolds numbers typical of small-scale UAV operations, ensuring efficient ISR loitering.

Stability Analysis

Static stability was a primary design requirement. The Center of Gravity (CG) was precisely calculated relative to the Mean Aerodynamic Chord (MAC) to ensure a positive static margin. This prevents pitch-instability and allows the Kalman filter to maintain smoother tracking during atmospheric disturbances.
